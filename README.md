このプロジェクトは、コマンドライン上で動作する単語当てゲームです。
入力された 1 文字を元に、単語内の一致箇所を表示し、制限回数内に全ての文字を当て。

## 環境構築

```bash
npm install
```

### ゲーム実行

```bash
npm start
```

### テスト実行

```bash
npm test
```

## 技術・ライブラリ

| カテゴリ | 技術・ライブラリ |
| -------- | ---------------- |
| 言語     | JavaScript       |
| 実行環境 | Node.js          |
| CLI 入力 | readline         |
| テスト   | Jest             |

## うまくできなかったこと。課題点

- TypeScript 化できてないです。型安全や補完が利用できていない。
- せっかくテスト書くならと思い初挑戦ではありましたが、TDD を意識して先に game.js 等のロジック単体とテストを実装したため、CLI 側でユーザー入力を受け取り、ゲーム状態を更新して表示する処理を繋ぐのが複雑になってしまいました。もっと責務をしっかり分離すべきだと思いました。
- 文字の入力に対するエラーハンドリングが少しだけしか出来ていないです。
